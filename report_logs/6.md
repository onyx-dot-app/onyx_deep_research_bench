## 1) 术语对齐：什么算“具身智能/Embodied AI”，什么算“VLA/机器人基础模型”

### 1.1 具身智能（Embodied AI）
具身智能强调智能体具备**物理本体**，能在真实环境中**主动感知—决策—行动**并与环境交互；机器人是最突出的具身形态之一，但具身智能也可扩展到智能家电/眼镜/车等载体[[1]](https://arxiv.org/html/2405.14093v6)。

### 1.2 VLA（Vision-Language-Action）与 LVLA
在语言条件机器人任务中，策略需要理解自然语言、感知环境并输出动作。VLA 泛指**多模态输入（视觉+语言）→ 输出机器人动作**的模型；典型结构是视觉编码器 + 语言/大模型 + 跨模态对齐 + 动作解码器/动作预测[[1]](https://arxiv.org/html/2405.14093v6)。RT-2 将“VLA”作为明确术语推广，并出现“Large VLA（LVLA）”等提法（类比 LLM）[[1]](https://arxiv.org/html/2405.14093v6)。

### 1.3 机器人基础模型（Robotics Foundation Model）
其目标是用**大规模、多样数据**训练出可迁移到新情境、能推理并快速学新任务的模型[[2]](https://arxiv.org/abs/2503.14734)。典型产业化表述是：通用机器人需要“通用身体 + 智能心智”，而基础模型就是“心智”的核心组成[[2]](https://arxiv.org/abs/2503.14734)。

## 2) 全球主流技术路线：从“端到端策略”到“平台化物理 AI 栈”

下面用多维框架先把路线族“定型”，再在每条路线下给出代表公司。

### 2.1 按系统架构划分的 8 条路线族（当前最主流）
> 这些路线族并非互斥；很多公司会“同时占多条路线”。

1. **端到端 VLA：动作 token 化/离散化，像语言一样预测动作**  
   代表架构：RT-2 把动作离散成 token 字符串，与 Web 视觉语言任务共同微调，直接从相机图像输出动作序列[[17]](https://deepmind.google/blog/rt-2-new-model-translates-vision-and-language-into-action/)。  
   典型价值：用互联网规模语义能力提升泛化与“初步推理”[[17]](https://deepmind.google/blog/rt-2-new-model-translates-vision-and-language-into-action/)。

2. **“慢思考规划 + 快系统控制”的双系统 VLA（常与扩散/生成式动作结合）**  
   代表架构：NVIDIA GR00T N1 双系统：System 2（VLM 推理/规划）+ System 1（扩散 Transformer 生成连续实时动作），紧耦合端到端训练[[2]](https://arxiv.org/abs/2503.14734)[[3]](https://nvidianews.nvidia.com/news/nvidia-isaac-gr00t-n1-open-humanoid-robot-foundation-model-simulation-frameworks)。  
   典型价值：把“推理与长上下文任务”与“高频连续控制”拆开做强。

3. **通用策略基础模型（Generalist Policy）：跨平台预训练 + 快速微调适配新本体/动作空间**  
   代表工作：Octo（开源通用策略）强调跨 9 种机器人平台、可在消费级 GPU 上数小时微调适配新观测与动作空间[[6]](https://arxiv.org/abs/2405.12213)。

4. **决策 Transformer/序列建模通用操控：多任务多本体 + 自改进数据回路**  
   代表工作：RoboCat（DeepMind）用 decision transformer 吸收跨模拟与真实的动作标注经验，支持零/少样本适配，并提出模型自生成数据的自我改进回路[[7]](https://arxiv.org/abs/2306.11706)。

5. **LLM/VLM 做高层规划器 + 低层技能库/价值函数落地（分层控制）**  
   代表工作：SayCan 把 LLM 的“技能序列建议”与价值函数的“可执行性/可供性”结合，选择既有用又可执行的技能并闭环执行[[9]](https://say-can.github.io/)。  
   代表工作：VoxPoser 用 LLM 推断约束与可供性、组合 3D value maps，再用模型式规划生成 6DoF 轨迹[[10]](https://arxiv.org/abs/2307.05973)。  
   注：VLA 综述也指出当前机器人系统常用这种“高层规划器 + 低层控制策略”的层级框架[[1]](https://arxiv.org/html/2405.14093v6)。

6. **合成数据/仿真到现实（Sim2Real）与“物理 AI 数据飞轮”**  
   NVIDIA 明确把 Omniverse + Cosmos + 蓝图（Teleop/Mimic/Dreams）组织成“数据生成—训练—回流”的飞轮式工作流，并宣称合成数据能显著扩大规模（但不少数字为公司口径）[[3]](https://nvidianews.nvidia.com/news/nvidia-isaac-gr00t-n1-open-humanoid-robot-foundation-model-simulation-frameworks)[[15]](https://developer.nvidia.com/isaac/gr00t)。  
   研究侧也有“Robot-powered data flywheel”：真实部署机器人收集数据、弱标注并持续微调基础模型以提升域内与域邻泛化（Scanford 例）[[13]](https://arxiv.org/abs/2511.19647)。

7. **世界模型/World Foundation Model（WFM）驱动的物理场景生成与预测（面向训练与评测）**  
   NVIDIA Cosmos 被定位为“世界基础模型平台”，含 Predict/Transfer/Reason 等模型，用于生成/预测物理世界视频与合成数据，加速机器人与自动驾驶等物理 AI 开发[[81]](https://www.nvidia.com/en-us/ai/cosmos/)。

8. **云端训练 + 端侧推理 / 边云协同部署（ROS2 生态与端侧算力平台并行演进）**  
   FogROS2 作为 ROS2 扩展，支持将 ROS2 计算图部分节点上云，自动配置云主机、加密通信、远程启动等[[87]](https://github.com/BerkeleyAutomation/FogROS2)[[88]](https://berkeleyautomation.github.io/FogROS2/cloud_robotics)；  
   端侧方向则以 Jetson Thor 等为代表，强调在机器人本体上运行多模态模型与功能安全能力（NVIDIA口径）[[16]](https://nvidianews.nvidia.com/news/foundation-model-isaac-robotics-platform)[[15]](https://developer.nvidia.com/isaac/gr00t)。

## 4) 把“公司”放回“技术路线”：典型技术栈对比（你可用于尽调/对标）

下面给出一套在尽调时最有用的“对比维度”，并把代表公司映射进去。

### 4.1 “大脑”路线：VLA/基础模型如何接控制闭环
- **动作 token 化端到端 VLA**：RT-2（DeepMind）把动作离散成 token 字符串共同微调[[17]](https://deepmind.google/blog/rt-2-new-model-translates-vision-and-language-into-action/)；Open X-Embodiment/RT-X 把跨本体数据规模化训练[[76]](https://deepmind.google/blog/scaling-up-learning-across-many-different-robot-types/)[[85]](https://robotics-transformer-x.github.io/)。  
- **双系统（推理规划 + 快速动作生成）**：GR00T N1 双系统 + 扩散 Transformer 动作模块[[2]](https://arxiv.org/abs/2503.14734)[[3]](https://nvidianews.nvidia.com/news/nvidia-isaac-gr00t-n1-open-humanoid-robot-foundation-model-simulation-frameworks)。  
- **连续动作生成式策略（flow matching/扩散变体）**：PI π0 用 flow matching 做高频连续控制[[19]](https://www.pi.website/blog/pi0)。  
- **跨形态通用控制（omni-bodied）**：Skild AI 用大量仿真身体多样性逼迫学习可迁移控制（公司自述）[[86]](https://www.skild.ai/blogs/omni-bodied)[[20]](https://www.skild.ai/blogs/series-c)。

### 4.2 数据路线：真实示范、遥操作、合成数据、世界模型如何组合
- **真实+合成数据飞轮（平台化）**：NVIDIA GR00T 工作流显式模块化 Teleop / Mimic / Dreams / Gen 等[[15]](https://developer.nvidia.com/isaac/gr00t)，并把 Omniverse + Cosmos 作为合成数据与世界生成引擎[[81]](https://www.nvidia.com/en-us/ai/cosmos/)[[82]](https://developer.nvidia.com/isaac/sim)。  
- **真实部署数据飞轮（以部署反哺模型）**：Robot-powered data flywheel（Scanford）展示两周部署采集数据、弱标注微调使识别大幅提升并带来域邻收益[[13]](https://arxiv.org/abs/2511.19647)；仓储侧 Covariant fleet learning 强调连接机器人持续学习[[78]](https://covariant.ai/covariant-brain/)；Amazon 获取 Covariant 基础模型许可并吸纳团队推进大规模运营应用[[22]](https://www.aboutamazon.com/news/company-news/amazon-covariant-ai-robots)。

### 4.3 工程化路线：云端训练/仿真与端侧推理

## 6) 各路线的“当前成熟度—未来 12/24 个月关键变量—潜在赢家特征”（给你做战略判断用）

### 6.1 端到端 VLA / 机器人基础模型（路线 1/2/3/4 的“核心大脑”）
**成熟度（2026Q1）**
- 研究成熟度快速提高：RT-2 展示 Web+机器人共同训练、动作 token 化带来的泛化与推理能力[[17]](https://deepmind.google/blog/rt-2-new-model-translates-vision-and-language-into-action/)；Open X-Embodiment/RT-X 在跨本体数据与训练上显著推进[[76]](https://deepmind.google/blog/scaling-up-learning-across-many-different-robot-types/)[[85]](https://robotics-transformer-x.github.io/)。  
- 工业平台化正在形成：NVIDIA 用 GR00T + Cosmos + Omniverse + Jetson 把“模型—数据—仿真—部署”串成全栈工具链[[15]](https://developer.nvidia.com/isaac/gr00t)[[81]](https://www.nvidia.com/en-us/ai/cosmos/)[[82]](https://developer.nvidia.com/isaac/sim)。  
- 商业公司融资极热：Skild AI、PI 等巨额融资说明资本押注“通用大脑”[[20]](https://www.skild.ai/blogs/series-c)[[32]](https://www.therobotreport.com/physical-intelligence-raises-600m-advance-robot-foundation-models/)。

**未来 12/24 个月关键变量**
1) **数据闭环效率**：是否能在真实部署中持续收集高价值数据并形成“可控、可扩展、合规”的后训练流程（Scanford 类型飞轮提供一种范式）[[13]](https://arxiv.org/abs/2511.19647)。  
2) **实时控制与安全**：高频控制（PI 的 flow matching/连续动作）、可解释与故障处理、与安全 PLC/工业标准集成（Agility AWS 案例提到与安全 PLC 集成）[[92]](https://aws.amazon.com/solutions/case-studies/agility-ai-case-study/)。  
3) **合成数据/世界模型的可用性**：Cosmos/Omniverse/Newton 这类工具若能显著降低 Sim2Real 成本，会决定很多团队的迭代速度[[81]](https://www.nvidia.com/en-us/ai/cosmos/)[[83]](https://developer.nvidia.com/blog/announcing-newton-an-open-source-physics-engine-for-robotics-simulation/)[[84]](https://isaac-sim.github.io/IsaacLab/main/source/experimental-features/newton-physics-integration/index.html)。  
4) **开源生态与标准化**：LeRobot 的数据格式与训练/评测脚手架，可能成为“PyTorch 时代的 ROS+数据集标准层”之一[[79]](https://github.com/huggingface/lerobot)[[80]](https://huggingface.co/blog/lerobot-datasets-v3)。

**潜在赢家特征**
- 既有模型能力，又掌握“数据—部署—运维”闭环（Amazon+Covariant、Covariant fleet learning）[[22]](https://www.aboutamazon.com/news/company-news/amazon-covariant-ai-robots)[[78]](https://covariant.ai/covariant-brain/)；  
- 或者掌握全栈平台入口（NVIDIA）[[15]](https://developer.nvidia.com/isaac/gr00t)[[82]](https://developer.nvidia.com/isaac/sim)；  
- 或者能在明确场景（仓储拣选、巡检）形成付费规模并反哺模型（Covariant、ANYbotics）[[78]](https://covariant.ai/covariant-brain/)[[70]](https://www.anybotics.com/news/anybotics-raises-additional-60-million-to-drive-u-s-expansion/)。

## 7) 我建议你下一步怎么用这份路线图（面向“你要做公司清单与对比表”的落地）

如果你的目标是做一份“可投/可合作/可采购”的全球清单，我建议用同一套表格模板把每家公司补齐到一致口径。基于本次来源现状，最需要补齐的字段是：

1) **商业化证据**：交付量、付费客户、合同/订单、定价（目前只有 1X、部分中国公司、Boston Dynamics Atlas 排期等较清晰）。  
2) **融资与估值一致口径**：Agility 的融资口径冲突需要用官方公告或权威数据库复核[[53]](https://research.contrary.com/company/agility-robotics)[[54]](https://www.ipo.club/deals/agility-robotics)[[55]](https://acquinox.capital/blog/agility-robotics-investor-insights)。  
3) **团队画像**：Skild AI、PI、1X、AgiBot 等的核心团队信息在本次来源覆盖不足。  
4) **技术栈可验证材料**：论文/开源代码/专利/第三方 benchmark。平台侧（NVIDIA、LeRobot）相对更容易获取；整机侧常需要客户试点报告与安全认证材料。

只要你告诉我：  
- 你希望覆盖的公司总数（例如 30/50/80 家），  
- 你更关注的场景（制造、仓储、巡检、家庭、医疗、零售等），  
- 以及你希望输出的格式（Excel 字段/数据库 schema/投委会 PPT），  
我可以在这份路线图之上，把“公司对比表”整理成可直接用于内部评审的结构化输出，并明确“证据等级/来源链接/待核验点”。
